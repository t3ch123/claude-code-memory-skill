# 项目记忆系统 — 跨会话长期记忆

## 一、本技能的作用

让 Claude Code 在不同会话之间保持记忆,记住:
- 上次做了什么
- 项目当前状态
- 未完成的待办
- 已知问题和决策

记忆文件保存在 .claude/memory/ 文件夹,内容用中文 markdown。

## 二、启动时必做(强制流程)

每次在本项目目录启动,第一件事:

1. 读取 .claude/memory/current_state.md
2. 读取 .claude/memory/todo.md
3. 读取 .claude/memory/known_issues.md
4. 浏览 .claude/memory/session_log.md 最近 3 条

读完后向用户汇报:

「我已读取上次记忆,简要回顾:
- 上次会话做了:XX
- 当前进度:XX
- 待办还有:1.XX  2.XX
- 已知问题:XX

你这次想继续做什么?」

⛔ 没读记忆就开始干活 = 严重违规

## 三、关键节点要更新记忆

| 触发场景 | 更新哪个文件 |
|---------|--------|
| 完成一个功能 | session_log.md + current_state.md + todo.md |
| 修复一个 bug | session_log.md + known_issues.md |
| 做了重要决策 | decisions.md |
| 发现新问题 | known_issues.md |
| 用户给了新需求 | todo.md |
| 改了关键配置 | current_state.md |

## 四、结束会话前必做

用户说"结束""再见""先到这"等关闭信号时,强制执行:

1. 总结本次会话,追加到 session_log.md
2. 更新 current_state.md
3. 更新 todo.md(完成的勾掉,新加的写上)
4. 向用户确认:「本次记忆已保存,下次启动会自动恢复」

⛔ 直接结束不保存 = 严重违规

## 五、记忆文件格式

### session_log.md 格式(追加式,最新放最上)

## 日期时间

**做了什么:**
- ...

**改了哪些文件:**
- ...

**需要重新打包:** 是/否

**遗留问题:** ...

---

### current_state.md 格式(覆盖式)

# 项目当前状态

## 最后更新
日期时间

## 已完成功能
- ...

## 进行中
- ...

## 未开始
- 见 todo.md

### todo.md 格式

# 待办事项

## 高优先级
- [ ] ...

## 中优先级
- [ ] ...

## 已完成(最近 7 天)
- [x] 日期 内容

### known_issues.md 格式

# 已知问题

## 待修复
- ...

## 已规避
- ...

## 已修复(可参考)
- 日期 问题 → 原因 → 解决

### decisions.md 格式

# 重要决策

## 日期
**决策:** ...
**原因:** ...
**影响:** ...

## 六、维护原则

1. 简洁 — 每条不超过 5 行
2. 可读 — 用人话,不要技术黑话
3. 时间戳 — 每条加日期
4. 可追溯 — 老记录不删,移到底部归档
5. 不存敏感信息 — 不要记密码、密钥、token

## 七、用户主动指令

| 用户说 | Claude 做 |
|------|------|
| "记一下:XXX" | 立即追加到合适文件 |
| "上次怎么改的 X" | 检索 session_log.md |
| "上次说要做什么?" | 读 todo.md |
| "清空记忆" | 备份后清空 |
| "导出记忆" | 打包 memory 文件夹 |
